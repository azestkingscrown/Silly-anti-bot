# Docker Deployment Guide

Quick Docker deployment of the Anti-Ad Bot v2.0

## Prerequisites

- Docker (20.10+)
- Docker Compose (2.0+)
- Discord Bot Token
- Server configuration (IDs)

## Quick Start

### 1. Create `.env` file from template

```bash
cp config/.env.example .env
```

Edit `.env` with your Discord settings:
```ini
DISCORD_TOKEN=your_token_here
GUILD_ID=123456789
MUTED_ROLE_ID=987654321
APPEAL_CHANNEL_ID=111111111
LOG_CHANNEL_ID=222222222
WEB_API_TOKEN=your-secure-token
```

### 2. Start Services

```bash
docker-compose up -d
```

Services:
- **anti-ad-bot** - Discord bot (port: internal)
- **anti-ad-web** - Web interface (port: 5000)

### 3. Access Web Interface

Open: `http://localhost:5000`

## Docker Commands

### View Logs
```bash
# Bot logs
docker-compose logs anti-ad-bot -f

# Web server logs
docker-compose logs anti-ad-web -f

# All logs
docker-compose logs -f
```

### Stop Services
```bash
docker-compose down
```

### Restart Services
```bash
docker-compose restart
```

### Rebuild Image
```bash
docker-compose up -d --build
```

### Check Service Status
```bash
docker-compose ps
```

## File Volumes

Persistent data locations:
- `Training-Data/` - Spam image samples
- `data.json` - User mutes and appeals
- `logs/` - Application logs

These are shared between containers and your host.

## Environment Variables

All variables from `config/.env` are supported:

| Variable | Required | Default |
|----------|----------|---------|
| DISCORD_TOKEN | Yes | - |
| GUILD_ID | Yes | - |
| MUTED_ROLE_ID | Yes | - |
| APPEAL_CHANNEL_ID | Yes | - |
| LOG_CHANNEL_ID | Yes | - |
| SIMILARITY_THRESHOLD | No | 0.75 |
| PUNISHMENT_TYPE | No | mute |
| WEB_API_TOKEN | No | change-this-token |

## Troubleshooting

### Container Won't Start
```bash
docker-compose logs anti-ad-bot
```
Check for missing environment variables or permission issues.

### Web Server Not Accessible
```bash
docker-compose ps
```
Ensure `anti-ad-web` is running and port 5000 is not blocked.

### Image Not Detected
Same as manual deployment - adjust `SIMILARITY_THRESHOLD` in `.env`

### Permission Denied on Volumes
Ensure training data and data.json are readable:
```bash
chmod 755 Training-Data/
chmod 644 data.json
```

## Advanced

### Run Only Bot (No Web)
Edit `docker-compose.yml` and remove `web-server` service, then:
```bash
docker-compose up -d
```

### Run Only Web Server
Ensure bot is running separately, then:
```bash
docker-compose up anti-ad-web -d
```

### Custom Port
Edit `docker-compose.yml`:
```yaml
web-server:
  ports:
    - "8080:5000"  # Host:Container
```

### Production Deployment

For production, consider:
1. Use secrets manager instead of `.env`
2. Add reverse proxy (nginx)
3. Enable HTTPS
4. Monitor container health
5. Set resource limits:

```yaml
services:
  anti-ad-bot:
    deploy:
      resources:
        limits:
          cpus: '1'
          memory: 512M
```
